# Testes Manuais

Casos de teste e relatórios de bugs documentados seguindo boas práticas de QA.

## Conteúdo
- **casos-de-teste/** — cenários escritos com pré-condições, passos e resultados esperados
- **relatorios-de-bug/** — bugs reportados com severidade, evidências e passos para reprodução
- **checklists/** — listas de verificação para smoke test e regressão rápida

## Padrão adotado
- IDs únicos por caso de teste (CT-XXX) e bug (BUG-XXX)
- Cobertura de cenários positivos, negativos e de validação
- Evidências anexadas sempre que possível
